# Skill: linus-review
description: Linus 式程式碼品味評分。快速三層判斷：品味評分、致命問題、改進方向。用於日常程式碼品質快速反饋。

## 執行流程

### 步驟 1：取得程式碼
若為檔案路徑則讀取，若為變更則取得 git diff。

### 步驟 2：三層判斷

**第一層：品味評分**
- 🟢 好品味：資料結構正確，無特殊情況，簡潔有力
- 🟡 湊合：能用但有改進空間，存在可消除的複雜性
- 🔴 垃圾：結構性問題，資料模型錯誤，過度複雜

**第二層：致命問題**
- 破壞 userspace（向後不相容）
- 資料結構根本性錯誤
- 安全漏洞（注入、XSS、敏感資料外洩）

**第三層：改進方向**
- 具體、可執行的建議，最多 3 項。

### 步驟 3：總結
提供整體判斷與最優先改進建議。
