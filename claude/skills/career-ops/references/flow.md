# Career-Ops 完整流程

## 資料目錄

所有資料存放於 `~/.claude/career-ops/`：

```
~/.claude/career-ops/
├── cv.md                 # 主履歷（Markdown 格式）
├── profile.yml           # 個人檔案：目標職位、偏好地區、薪資範圍、技能
├── article-digest.md     # 選填：文章、演講、專案成果等 proof points
├── stories/              # STAR+R 面試故事庫
│   └── *.md              # 每個故事一個檔案
├── pipeline.md           # 求職 pipeline 總覽
└── resumes/              # 產出的客製化履歷
    └── {company}-{role}-{date}.md
```

## 子指令

### `/career-ops init`

初始化資料目錄，建立模板檔案。

**流程：**
1. 建立 `~/.claude/career-ops/` 及子目錄
2. 若 `cv.md` 不存在，建立模板（提示使用者填入）
3. 若 `profile.yml` 不存在，建立模板
4. 若 `pipeline.md` 不存在，建立空表格
5. 報告初始化結果

### `/career-ops evaluate <url>`

評估單一職缺，產出 A-F 評分。

**流程：**
1. 用 `WebFetch` 抓取職缺頁面內容
2. 讀取 `cv.md` 和 `profile.yml`
3. 依據 10 個加權維度評分（見下方評分系統）
4. 產出評估報告，含：
   - 總分（A-F）
   - 各維度分數
   - 匹配分析（你的經歷 vs JD 要求）
   - 建議：值得投 / 考慮 / 跳過
5. 詢問是否加入 pipeline

**10 維度評分系統（加權）：**

| # | 維度 | 權重 | 說明 |
|---|------|------|------|
| 1 | 職位匹配度 | 20% | 你的經歷與 JD 核心要求的吻合度 |
| 2 | 技能匹配度 | 15% | 必備技能 vs 你的技能清單 |
| 3 | 成長潛力 | 10% | 這份工作對你職涯發展的價值 |
| 4 | 公司規模/階段 | 10% | 與你偏好的公司類型是否匹配 |
| 5 | 地點/遠端 | 10% | 與你的地點偏好是否匹配 |
| 6 | 薪資預期 | 10% | JD 薪資範圍 vs 你的期望（若有揭露） |
| 7 | 團隊/管理 | 5% | 匯報對象、團隊規模、文化線索 |
| 8 | 技術棧 | 10% | 使用的工具/語言是否你熟悉或想學的 |
| 9 | JD 品質 | 5% | JD 寫作品質、是否清楚描述期望 |
| 10 | 紅旗偵測 | 5% | 是否有可疑訊號（超長要求、不合理期望等）|

**評分等級：**
- **A** (90-100)：高度匹配，優先投遞
- **B** (75-89)：不錯的機會，值得投
- **C** (60-74)：普通匹配，考慮投
- **D** (45-59)：匹配度低，除非有特殊理由
- **F** (0-44)：不匹配，跳過

### `/career-ops resume <url-or-id>`

為特定職缺產出客製化履歷。

**流程：**
1. 取得 JD 內容（從 URL 抓取，或從 pipeline 中找已評估的職缺）
2. 讀取 `cv.md`、`profile.yml`、`article-digest.md`（若有）
3. 分析 JD 關鍵要求
4. 客製化策略：
   - 挑出最匹配的經歷，重新排列優先順序
   - 針對 JD 關鍵字調整用詞（ATS 優化）
   - 量化成果（數字優先）
   - 移除不相關經歷或降低其篇幅
5. 產出客製化履歷 Markdown，存入 `resumes/`
6. 顯示 diff：標出與原始 cv.md 的差異重點

**注意：**
- 不會發明經歷或誇大成果
- 只做「選擇」和「重排」，不做「捏造」
- 每次產出都標註基於哪個 JD

### `/career-ops stories [add|list|match <url>]`

管理 STAR+R 面試故事庫。

**子指令：**

- `stories add` — 互動式引導建立新故事
  - Situation：背景是什麼？
  - Task：你的任務/挑戰是什麼？
  - Action：你採取了什麼行動？
  - Result：成果是什麼？（量化）
  - Reflection：學到什麼？下次會怎麼做？
  - Tags：相關技能/領域標籤

- `stories list` — 列出所有故事，按標籤分類

- `stories match <url>` — 讀取 JD，推薦最匹配的故事
  - 分析 JD 可能的行為面試問題
  - 從故事庫配對最適合的故事
  - 標出缺口：哪些面試題目前沒有對應故事

### `/career-ops pipeline [add|list|update]`

追蹤求職進度。

**Pipeline 欄位：**

```markdown
| 公司 | 職位 | 評分 | 狀態 | 投遞日期 | 下一步 | 備註 |
```

**狀態流：**
`評估中` → `準備投遞` → `已投遞` → `電話面試` → `技術面試` → `最終面試` → `Offer` / `拒絕` / `放棄`

- `pipeline add <company> <role> <grade>` — 新增條目
- `pipeline list [--status <status>]` — 列出（可篩選狀態）
- `pipeline update <company> <status> [notes]` — 更新狀態

### `/career-ops` (無子指令)

顯示 dashboard：
1. Pipeline 概況（各狀態數量）
2. 最近評估的職缺
3. 故事庫統計
4. 提醒：需要跟進的項目

## 台灣適用性注意事項

- 直接支援使用 Greenhouse、Ashby、Lever、Workable 等歐美 ATS 的公司
- 104/CakeResume 等本土平台：WebFetch 抓取後可能需要手動貼入 JD 文字
- 在台外商：通常使用歐美 ATS，可直接用 URL
- 建議 profile.yml 中用英文填寫（JD 比對更準），備註可用中文

## 重要原則

1. **不自動投遞** — 只做評估和準備，投遞由使用者自行決定
2. **不發明經歷** — 客製化是「選擇」和「重排」，不是「捏造」
3. **隱私優先** — 所有資料留在本地 `~/.claude/career-ops/`，不外傳
4. **漸進學習** — 系統越用越了解你，前幾次產出需要多回饋調整
