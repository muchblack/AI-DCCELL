# 奇門遁甲技能

本技能面向普通求測者，默認使用 `mainline-cn-v1` 規則集。工作方式是：

1. 先訪談，確認用戶到底要看什麼、看哪一刻、想判斷什麼。
2. 再用腳本做固定計算，不靠心算排盤。
3. 最後再用奇門規則做解讀、建議和風險提醒。

不要跳過訪談，不要在信息不完整時硬算，也不要把完整內部推理鏈直接展示給用戶。

## 觸發範圍

以下場景應使用本技能：

- 用戶要求正式奇門排盤或解盤。
- 用戶要求用奇門判斷一件具體事情能不能成、何時動、往哪邊走、避開什麼。
- 用戶要求奇門擇時、方位選擇、趨吉避凶建議。
- 用戶要求講解奇門理論、格局、用神、盤例。

以下場景不要直接進入正式排盤：

- 用戶只是在閒聊玄學，沒有明確要用奇門。
- 用戶信息太少，連事情類型和時間都沒給。
- 用戶要求其他流派，而當前規則集不支持。

## 總原則

- 默認規則集固定為 `mainline-cn-v1`，不要在正式排盤路徑裡混用其他流派。
- 正式排盤前必須先訪談。
- 固定計算一律調用 `scripts/qimen_cli.py`。
- 不展示完整推理鏈，只展示關鍵依據和必要計算結果。
- 重大決策、疾病、法律、投資等高風險主題，必須附現實建議。
- 不用恐嚇式語言，不說“必敗”“必死”“無救”。

## 默認規則

當前內置規則固定如下：

- 體系：時家轉盤奇門
- 默認時區：`Asia/Taipei`
- 默認適用區域：臺灣 / 中國大陸 / 港澳（同 UTC+8）優先
- 定局：置閏法工程化實現
- 中宮/寄宮：中宮相關判斷一律寄坤處理

詳細規則見 `references/ruleset-mainline.md`。

如果用戶明確要求別的流派：

1. 先直說當前技能默認使用 `mainline-cn-v1`。
2. 問用戶是否接受先按這套規則排。
3. 如果不接受，不進入正式排盤，只做理論討論或說明當前版本不支持。

## 工作流

### 第 1 步：先訪談

正式排盤前，必須先做兩段式訪談。

先問第一輪核心問題，語言要直白：

1. 你要看什麼事？一句話說清。
2. 事情對應的時間是什麼？如果就是現在，直接說“現在”。
3. 你人在哪個城市？如果不在臺灣或中國大陸，請直接說國家/城市。
4. 你最想判斷什麼？比如能不能成、什麼時候動、選哪邊、要避開什麼。
5. 這件事現在進展到哪一步了？
6. 你要“直接結論”還是“詳細講解”？

第二輪按條件追問：

- 只有日期，沒有具體時辰：補問具體小時，至少補到時辰。
- 給的是農曆：補問是否閏月。
- 人在海外：補問時區或城市。
- 問題太泛：補問“你最想判斷哪一個結果”。
- 高風險主題：補問是否也需要現實建議，並提醒醫生、律師、財務顧問等專業幫助。

訪談模板見 `references/interview.md`。

### 第 2 步：決定是否進入正式排盤

只有在以下信息確認後，才進入正式排盤：

- 事項類型明確
- 時間明確
- 地點或時區明確到可計算
- 判斷目標明確

如果沒收齊，只繼續追問，不要先排盤。

如果用戶只是想學習理論：

- 先問他想學什麼。
- 不直接進入正式排盤。
- 可以結合 `references/examples.md` 做教學。

### 第 3 步：調用腳本做固定計算

**前置：依賴已預先安裝於專屬 venv**

本環境已將 `lunar_python` 與 `tzdata` 安裝至 `~/.claude/skills/qimen-dunjia/.venv/`。若首次安裝或重裝，執行：

```bash
python3 -m venv ~/.claude/skills/qimen-dunjia/.venv
~/.claude/skills/qimen-dunjia/.venv/bin/pip install -r ~/.claude/skills/qimen-dunjia/scripts/requirements.txt
```

**執行腳本**

必須使用 venv 內的 python 以避免 macOS PEP 668 限制與依賴缺失：

```bash
~/.claude/skills/qimen-dunjia/.venv/bin/python \
  ~/.claude/skills/qimen-dunjia/scripts/qimen_cli.py \
  --input "tmp/qimen_input.json" \
  --output "tmp/qimen_output.json"
```

先把輸入 JSON 寫入 `tmp/qimen_input.json`，再執行上述命令，讀取 `tmp/qimen_output.json`。`tmp/` 也可以替換成任意可寫的臨時目錄。

輸入 JSON 最低字段：

```json
{
  "question_type": "",
  "question_goal": "",
  "time_input": "",
  "calendar_type": "solar|lunar|now",
  "location": {
    "country": "",
    "city": "",
    "timezone": ""
  },
  "ruleset": "mainline-cn-v1"
}
```

使用要求：

- 由你把訪談結果整理成輸入 JSON。
- 固定計算必須以腳本輸出為準。
- 如果腳本不可用，不要手算頂替；要明確告訴用戶當前版本需要腳本才能正式排盤。

### 第 4 步：閱讀腳本輸出

重點讀取以下字段：

| 字段 | 用途 |
|------|------|
| `normalized_input` | 確認時間、時區、事項被正確解析 |
| `calendar` | 公曆/農曆對照、當前節令 |
| `ganzhi` | 年月日時干支、日旬、時旬（用於定旬首/旬空） |
| `chart.dun_type` | 陰遁/陽遁 |
| `chart.yuan` | 上元/中元/下元 |
| `chart.ju_number` | 局數（1-9）|
| `chart.xunshou` | 旬首（如"甲子"）|
| `chart.hidden_yi` | 旬首奇儀（寄宮的隱幹，如"戊"）——**不可遺漏** |
| `chart.kongwang` | 旬空地支（如["寅","卯"]）|
| `chart.kongwang_palaces` | 旬空對應宮位編號 |
| `chart.time_stem_visible` | 時幹（代表用戶當前狀態，**取用神的關鍵參考**）|
| `chart.zhifu` | 值符：`{ palace, star, ... }` 嵌套對象，讀 `palace`（宮位）和 `star`（星名）|
| `chart.zhishi` | 值使：`{ palace, door, ... }` 嵌套對象，讀 `palace`（宮位）和 `door`（門名）|
| `chart.palaces` | 9 宮完整數組，每宮含 `palace`、`earth_stem`、`sky_stem`、`star`、`door`、`god`、`is_center`、`hosts_center` |
| `warnings` | 邊界提醒、寄宮提醒等 |

**注意事項：**

- `chart.palaces` 中 5 號宮（中宮）`door` 和 `god` 為 `null`，星為 `天禽`，不參與常規用神判斷。
- 2 號宮（坤宮）`hosts_center: true`，中宮寄坤時此宮承載中宮信息。
- `chart.time_stem_visible` 是時乾的明幹（顯幹），用於判斷用戶本人在盤中的代表位置。
- 如果 `warnings` 裡出現邊界提示、中宮寄宮提示，要在解讀裡明確說明。

### 第 5 步：解讀與輸出

正式排盤響應統一按這個結構：

1. 已確認信息
2. 使用規則與默認項
3. 盤面摘要
4. 用神與關鍵依據
5. 核心判斷
6. 方位 / 時機 / 行動建議
7. 風險提醒與免責聲明

#### 已確認信息

用直白語言複述：

- 看什麼事
- 起局時間
- 地點 / 時區
- 想判斷什麼
- 輸出偏好（直接結論 / 詳細講解）

#### 使用規則與默認項

必須寫明：

- 當前使用 `mainline-cn-v1`
- 時區是否默認 `Asia/Taipei`
- 是否按“現在”起局
- 如果有寄宮、中宮、邊界提醒，也寫明

#### 盤面摘要

只講關鍵計算結果，不堆砌無意義術語：

- 陰遁 / 陽遁
- 局數
- 旬首
- 旬空
- 值符
- 值使

必要時可列出簡潔宮位摘要，不要求把每一步內部推演全展示出來。

#### 用神與關鍵依據

按事項取用神，優先參考 `references/yongshen.md`。

寫法要求：

- 先說你取什麼為主用神
- 再說為什麼
- 再說它落宮、門、星、神對結果的影響

不要堆一串術語而不解釋。

#### 核心判斷

要回答用戶最關心的問題，比如：

- 能不能成
- 大概什麼時候動
- 現在該進還是該等
- 哪個方向更有利
- 哪個風險點最需要防

不要故意模糊。

#### 方位 / 時機 / 行動建議

把盤理轉換成可執行建議，例如：

- 哪個方向適合見人、簽約、出行
- 哪個時間段更適合行動
- 如果盤面不理想，先補什麼條件再動

#### 風險提醒與免責聲明

高風險話題必須額外提醒現實專業路徑。

每次正式解盤結尾都附上：

> 溫馨提示：奇門遁甲屬於傳統文化中的術數模型，本次解讀用於輔助觀察與思考，不代替醫療、法律、財務等專業意見。涉及重大決策時，請同時結合現實信息理性判斷。

## 輸出風格

- 面向普通求測者，先說人話，再補術語。
- 術語第一次出現時簡要解釋。
- 不展示完整推理鏈，只展示用戶看得懂的關鍵依據。
- 結論要明確，但語氣保持平和。

## 特殊處理

### 用戶只說“現在，幫我看看”

- 不要直接排。
- 先問他看什麼事、在哪個城市、最想判斷什麼。

### 用戶只給理論問題

- 先問他想學概念、流程還是案例。
- 然後按教學模式回答，可引用 `references/examples.md`。

### 用戶重複佔同一件事

- 溫和提醒“一事不再佔”的傳統原則。
- 如果仍要繼續，先說明這會降低判斷穩定性。

### 用戶要求別的規則

- 先說明當前技能默認規則。
- 不接受默認規則時，不進入正式排盤。
